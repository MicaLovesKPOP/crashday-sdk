This is a short overview on all the modding-relevant file types used by Crashday.

.amb

.amb files are used to define ambiences. These are just plain text files

ladder.lst

ladder.lst found in career folder is a plain text defining enemies names in career

.lst

Plain text file. Defines parts that can be purchased in the career shop

.crv

Plain text file. Engine torque curve file describing the car engine's torque behaviour

.cca

Plain text file. Main car setup file containing all of the vehicles parameters

.tun

Plain text file. Lists all tuning parts of the car available

engsound.txt

Plain text file. Engine sound definition of the car

.dbs

Plain text file. Contains game's parameters

.cdo

Plain text file. Defines dynamic object's parameters

.cat

Plain text file. Category divisions for tiles available in the editor

ctable.cnv

Plain text file. Font conversion table

fontinfo.dbs

Plain text file. Font definition file

.wid

Binary file containing the width for each character in the font texture

.p3d

Binary model file

.sys

Plain text file. Defines particle's texture and properties

.tex

Plain text file. Texture's shader definition

.cai

Plain text file. Defines AI nodes on the spicific tile

.cfl

Plain text file. Tile definition file

.cst

Plain text file. Tile's stunt definitions

.veg

Plain text file. Defines vegetation and it's properties

.whl

Plain text file. Wheel definition file.

.cpk

content package archive containing data and information. Can be opened with any archiver (E.g. winRAR, 7zip)

.amb, .cca, .sys are different from original Crashday release. They will not work unless remade corresponding to new file structure.
